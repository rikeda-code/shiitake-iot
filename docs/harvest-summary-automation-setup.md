# 収穫実績集計表 新規タブ自動追加 セットアップ手順

「号機別収穫実績集計表」スプレッドシートの「収穫実績集計表1」タブで、
号機ごとのタブ(例:「16(7/13)L3」)を新規追加すると、自動で以下を行います。

1. 3行目(見本行)の下に1行追加
2. 3行目にある関数(A~Q列)と同じ関数を追加された4行目に展開
3. 追加された4行目のF列に新規タブ名を記入

## 前提: 既存のApps Scriptコードを確認する

このスプレッドシートに既に他のApps Scriptコードが入っている場合、
上書きすると既存の自動化が壊れる可能性があります。まず「拡張機能」→
「Apps Script」を開き、既存コードの有無を確認してください。

- 何も書かれていない場合 → そのまま貼り付けてOK
- 既存コードがある場合 → 既存コードは残したまま、`dashboard/harvest-summary-automation.gs`
  の中身をその下に追記してください(Claude Codeに「このスプレッドシートの
  Apps Scriptに追記して」と伝えれば確認しながら進められます)

## 手順

1. スプレッドシート「号機別収穫実績集計表」を開く
2. メニュー「拡張機能」→「Apps Script」を開く
3. リポジトリの `dashboard/harvest-summary-automation.gs` の中身を貼り付けて保存する
4. 関数選択のプルダウンから `createOnChangeTrigger` を選び、実行ボタン(▶)をクリック
   - 初回は権限の承認を求められるので許可する
   - これでタブの追加・リネームを検知するトリガーが1回だけ登録されます(再実行しても重複登録はされません)
5. スプレッドシートを再読み込みすると、メニューバーに「収穫実績集計 自動化」が追加されます

## 使い方

1. いつも通り、新しいロットのタブを手動で追加する(例: 既存タブをコピーして
   リネーム、または「+」で新規タブを作成してリネーム)
2. タブ名を確定させた時点(リネームが完了した瞬間)で自動的に
   「収穫実績集計表1」の3行目の下に1行追加され、関数が展開され、
   F列に新しいタブ名が入ります
3. 自動で反映されない場合は、対象タブを開いた状態でメニュー
   「収穫実績集計 自動化」→「今開いているタブを集計表に追加」を実行してください
   (既に追加済みのタブは二重に追加されないようになっています)

## 注意点

- 行が追加されるのは、タブ名が **「号機番号(月/日)◯◯L数字」** の形式
  (例:「16(7/13)L3」「32(6/29)6志摩L3」)に一致する場合のみです
  (`LOT_NAME_PATTERN`)。「token」「test」「17pp」「15（池田）」のような
  管理用・個人用・テスト用のタブは、この形式に一致しないため自動的に対象外になります
- 上記パターンに一致してしまうがロットタブではないタブ名がある場合は、
  `dashboard/harvest-summary-automation.gs` 内の `EXCLUDED_SHEET_NAMES` に
  そのタブ名を追記してください
- シートを複製した直後の仮の名前(「◯◯のコピー」「シート2」など)は無視され、
  リネームして最終的なタブ名が確定した時点で処理されます
- 3行目(見本行)の関数自体は変更していません。現在3行目に `#REF!` などの
  エラーが出ている場合、そのまま4行目にも展開されるため、必要であれば
  3行目の関数を別途修正してください
- メニューの「収穫実績集計 自動化」→「今開いているタブを集計表に追加」は
  手動フォールバックのため、上記の命名パターンに一致しないタブでも実行できます
  (意図的に例外的なタブを追加したい場合のみ使用してください)

## 既に追加されてしまった不要な行を削除する

今回の修正前のバージョンでは、「token」「test」「15（池田）」のような
ロットタブではないタブの行も「収穫実績集計表1」に追加されてしまっていました。
不要な行は以下の手順で削除してください。

1. 「収穫実績集計表1」タブでF列を確認し、ロット名として不自然な行
   (「token」「test」「17pp」「群馬管理原本」「シート名リスト」
   「月稼働コンテナ内」「収穫実績集計表」「◯（池田）」「◯（小林）」など)を探す
2. 該当する行番号を左端でクリック(複数ある場合はCtrlキーを押しながら追加でクリック)
3. 選択した行番号の上で右クリック→「選択した行を削除」
4. 更新後のスクリプトを再度貼り付け・保存しておけば、今後は同じ問題は起きません
