# 社用車管理アプリ セットアップ手順

`dashboard/vehicles.html` は「安中ファーム 走行日報」スプレッドシートを読み込んで、
車両ごとの走行距離・車検・オイル交換・タイヤ交換を表示します。

- 走行距離や車検・交換記録の**閲覧**は、スプレッドシートの共有設定(社内ドメインに閲覧許可)だけで動きます。追加設定は不要です。
- 車検満了日・オイル交換・タイヤ交換の**入力(書き込み)** だけは、Google Apps Script (GAS) の
  Web Appを1回デプロイする必要があります。以下の手順で設定してください(社内のGoogleアカウント権限が必要な作業です)。

## 手順

1. スプレッドシート「安中ファーム 走行日報」を開く
2. メニュー「拡張機能」→「Apps Script」を開く
3. デフォルトで開くコードを全て削除し、リポジトリの `dashboard/vehicle-maintenance.gs` の中身を貼り付けて保存する
4. 右上の「デプロイ」→「新しいデプロイ」を選択
5. 種類の選択で「ウェブアプリ」を選ぶ
   - 「次のユーザーとして実行」: 自分
   - 「アクセスできるユーザー」: 全員(または社内のみ、必要に応じて選択)
6. 「デプロイ」をクリックし、初回は権限の承認を求められるので許可する
7. 発行された「ウェブアプリのURL」をコピーする
8. コピーしたURLを `dashboard/vehicles.html` 内の

   ```js
   const MAINTENANCE_API_URL = '';
   ```

   の `''` の部分に貼り付けて保存・反映する(Claude Codeに「このURLを設定して」と伝えればOK)

これで「車検満了日」「オイル交換日/走行距離」「タイヤ交換日/走行距離」を
アプリ画面から保存できるようになります。保存されたデータは自動的に作成される
「車両マスタ」というシート(タブ)に記録され、どのデバイス・どのアカウントから見ても
同じ内容が反映されます。

## データの仕組み

- 走行距離: 既存の「安中ファーム」シート(走行日報のログ)から、各車両の最新の報告を自動集計
- 車検満了日・オイル交換・タイヤ交換: 新設される「車両マスタ」シートに保存(アプリの保存ボタンから更新)

## 交換目安について

オイル交換・タイヤ交換の「交換目安超過」バッジは、`dashboard/vehicles.html` 内の
以下の定数で調整できます(初期値は一般的な目安です)。

```js
const OIL_INTERVAL_KM = 5000;
const OIL_INTERVAL_DAYS = 180;
const TIRE_INTERVAL_KM = 30000;
const TIRE_INTERVAL_DAYS = 1460;
```
